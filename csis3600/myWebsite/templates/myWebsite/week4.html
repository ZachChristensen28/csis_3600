1. Differences between internal and external fragmentation. For each of the four memory management systems described in class, identify which causes each type of fragmentation

Internal Fragmentation
Internal Fragmentation deals with fixed sizes of memory
External Fragmentation
Dynamic sized memory
Single User
Single user system has only one program area which only one program can run at a time.
Fixed
Allows for multiprogramming but has a fixed size which can cause internal fragmentation. This also leads to lots of wasted memory space.
Dynamic
Memory is allocated on a first come first serve basis. Problem is that this can cause external fragmentation when some jobs finish before others.
Relocatable dynamic
When fragmentation occurs, all jobs halt. unused space is allocated at the bottom of memory. All programs are then restarted.

2. Which type of fragmentation is reduced by compaction.

Internal Fragmentation. When memory becomes to fragmented, compaction is necessary to free up memory space.

3. How often should relocation be performed?

Memory relocation should be performed when memory becomes too fragmented

4. Give at least three negative outcomes that would result if one of these early operating systems did not perform memory compaction.

  1. Programs would have to wait longer until they are able to execute
