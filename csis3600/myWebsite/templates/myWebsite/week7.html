1. Percisely Define the following terms:

Access time
time delay between the request and delivery (data return, process executes, etc.)
Buffer
temporary storage location used in the movement of data
Direct memory access
components that allow I/O modules to directly read and write data in memory without having to travel through the CPU.
Device Driver
controls an external device giving the OS an interface to work with
Device Manager
feature of the OS where you are able to view and manage devices that are connected to the computer
ramdisk
virtual memory taken from the physical RAM. Created and used for quick access times
Single bus architecture
I/O devices, Processor and Memory are all located on the same bus. Are all easily able to access one another like another memory location.
Bus Controller
handles sending and receiving commands for the CPU
Memory word
the register size. The smallest size unit that can be addressed in memory.
clock cycle
time between two pulses of an oscillator
I/O controller
interfaces between the CPU and the external hardware device
charcter devices
driver communicates with this device sending and receiving characters such as bytes or octets
block devices
driver communicates with this device by sending blocks of data
sockets
Used for data go in and out of the systems network and managing all the network protocol that goes with it. Sockets are an I/O abstraction and establish connection with remote machines.

3. Describe how primay storage differs from secondary storage

Primary storage resides where the operating system does. There are direct buses from the CPU to that storage center making access time quick. Primary storage is temporary. Secondary storage is located outside of the CPU and must be connected through a data cable. It is not directly accessible by CPU therefore access time is longer. Secondary storage is permanent. In software development you want to restrict the disk access.
