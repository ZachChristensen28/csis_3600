Problem: 1
There is no limit to the number of processes that can be in the ready and blocked states. At most, n processes may be in the run state, since a process in the run state must be allocated to a CPU, and there are only n CPUs.
Problem: 2
There can be zero processes in any of the three states. It is possible for all the processes to be blocked while waiting on I/O operations, leaving no processes in the ready and running states. All processes can be either in the ready or run state, leaving no processes blocked.

List the Linux commands we covered today and what they do

  • chown

    Change ownership of a file

  • find

    Searches the directroy tree

  • cp

    Copies Files

  • vmstat

    Provides a snapshot of everything in a system

  • df

    Reports filesystem disk space usage

  • grep

    Used for searching one or more files

  • cpio

    Back up empty directories

  • alias

    Allows substitution of a small or more familiar name in place of a long string

  • awk

    Searches for patterns in a file and processes them

  • wc

    Counts words

  • diff

    Shows the differences between files

  • less

    Lest you scroll through files

  • set

    Show all environment variables

  • export

    Sets the value of a variable so it is visible to all sub-process that belong to the current shell
